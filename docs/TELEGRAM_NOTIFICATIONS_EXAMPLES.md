# 📱 Примеры Telegram уведомлений

Этот документ содержит примеры всех возможных Telegram уведомлений, которые отправляются при выполнении Fastlane lanes.

## 🔄 Режим прогресс-сообщения

В режиме прогресс-сообщения (`TELEGRAM_PROGRESS_MODE=true`) одно сообщение обновляется по мере выполнения этапов.

### Пример: Полный workflow с прогресс-сообщением


## 🔄 Стадии процесса

### Начало сборки проекта

**Когда отправляется:** В начале процесса сборки, после валидации переменных окружения

**Пример уведомления:**

```
🔄 Начало сборки приложения
━━━━━━━━━━━━━━━━━━━━

🎯 AppName • v1.2.3 • b456

🧭 Контекст
📱 Версия: 1.2.3
🔢 Build: 456
🛣️ Lane: build
• Target: AppName

🔗 Открыть Pipeline
```

**Код отправки:**
```ruby
notify_telegram_stage('Начало сборки приложения', {
  'Версия' => ENV['APP_VERSION'],
  'Build' => BUILD_NUMBER.to_s,
  'Target' => MAIN_TARGET.to_s
})
```

**Особенности:**
- Отправляется сразу после валидации переменных окружения
- Содержит базовую информацию о версии, build и target
- Позволяет отслеживать начало процесса сборки

### Сборка завершена успешно

**Когда отправляется:** После успешной сборки IPA файла

**Пример уведомления:**

```

🔢 Build: 456
🛣️ Lane: build
• Target: AppName

📦 Артефакты

📄 IPA файл: App_v1.2.3_b456.ipa
📄 dSYM: App.app.dSYM.zip
📦 Ipa size mb: 125.5
📦 Ipa size gb: 0.123

⏱️ Время
⏱️ Время выполнения: 5 мин 23 сек
⏱️ Duration (sec): 323.45

➕ Дополнительно
• Status: success

🔗 Открыть Pipeline
```

**Код отправки:**
```ruby
notify_telegram_success('Сборка завершена успешно', {
  'Версия' => version_number,
  'Build' => build_number.to_s,
```

**Особенности:**
- **Автоматически включает метрики** выполнения (время, размеры файлов)
- Показывает время выполнения этапа

## ❌ Ошибки

### Ошибка выполнения lane

**Когда отправляется:** При возникновении ошибки в любом lane

**Пример уведомления:**

```
❌ Ошибка выполнения lane
━━━━━━━━━━━━━━━━━━━━

🎯 AppName • v1.2.3 • b22

🧭 Контекст
🛣️ Lane: build
• Target: AppName
• Проект: mobile/ios/hockey
• Время: 2024-01-15 14:30:25

❌ Ошибки
❌ Ошибка: IPA файл не найден после сборки: ~/.cache/apps/Hockey/ipa/Hockey_v1.0.0_b22.ipa
❌ Тип ошибки: FastlaneCore::Interface::FastlaneError

⏱️ Время
⏱️ Время до ошибки: 3 мин 12 сек

🔗 Открыть Pipeline
```

**Код отправки:**
```ruby
notify_telegram_error("Ошибка выполнения lane", exception, {
  'Lane' => lane.to_s,
  'Target' => MAIN_TARGET.to_s,
  'Время' => Time.now.strftime('%Y-%m-%d %H:%M:%S'),
  'Проект' => ENV['CI_PROJECT_NAME'] || ENV['GITHUB_REPOSITORY'] || 'не указан'
})
```

**Особенности:**
- Сообщение об ошибке обрезается до 200 символов для читаемости
- Автоматически добавляется тип ошибки
- Показывается время выполнения до момента ошибки

## 🔍 Детали форматирования

### Эмодзи для полей

Система автоматически определяет эмодзи на основе названия поля:

- 📱 **Версия/Version** — для версии приложения
- 🔢 **Build/Сборка** — для номера сборки
- 📦 **Размер/Size** — для размеров файлов
- 📄 **IPA/Файл/File** — для файлов
- 👥 **Группы/Groups** — для групп тестировщиков
- ⏱️ **Время/Time/Duration** — для времени выполнения
- 🛣️ **Lane** — для названия lane
- ❌ **Ошибка/Error** — для ошибок
- • **Остальные поля** — стандартная точка

### Форматирование значений

- **Числа (Float)** — автоматически округляются до 2 знаков после запятой
- **Массивы** — автоматически объединяются через запятую
- **Строки** — выводятся как есть
- **Значения в `<code>`** — отображаются моноширинным шрифтом

### Ссылка на Pipeline

Ссылка автоматически добавляется, если доступна одна из переменных окружения:

**GitLab CI:**
- `CI_PIPELINE_URL` — прямая ссылка на pipeline
- `CI_JOB_URL` — ссылка на конкретный job
- `CI_PROJECT_URL` + `CI_PIPELINE_IID` — формируется ссылка на pipeline

**GitHub Actions:**
- `GITHUB_SERVER_URL` + `GITHUB_REPOSITORY` + `GITHUB_RUN_ID` — формируется ссылка на run

**Jenkins:**
- `BUILD_URL` — прямая ссылка на build
- `JOB_URL` — ссылка на job

## ⚙️ Настройка

Для включения Telegram уведомлений установите переменные окружения:

```bash
export TELEGRAM_ENABLED="true"
export TELEGRAM_BOT_TOKEN="123456789:ABCdefGHIjklMNOpqrsTUVwxyz"
export TELEGRAM_CHAT_ID="123456789"
```

### Отключение метрик в финальных уведомлениях

По умолчанию метрики автоматически включаются в финальные уведомления. Если нужно отключить:

```ruby
notify_telegram_success('Сборка завершена', {
  'Версия' => '1.2.3'
}, include_metrics: false)
```

### Отключение ссылки на pipeline

Если нужно отключить ссылку на pipeline в конкретном уведомлении:

```ruby
notify_telegram_success('Сборка завершена', {
  'Версия' => '1.2.3'
}, include_pipeline_link: false)
```

## 📝 Примечания

1. **Автоматическое добавление данных:**
   - Время выполнения добавляется автоматически из метрик
   - Название lane добавляется автоматически из метрик
   - Метрики включаются автоматически в финальные уведомления
   - Ссылка на pipeline добавляется автоматически, если доступны переменные CI/CD

2. **Ограничения:**
   - Сообщения об ошибках обрезаются до 200 символов
   - Telegram имеет лимит на длину сообщения (4096 символов)

3. **Поддержка HTML:**
   - Все уведомления используют HTML форматирование
   - Поддерживаются теги: `<b>`, `<code>`, `<a>`

4. **Метрики:**
   - Метрики автоматически включаются в финальные уведомления
   - Отдельная отправка метрик через `finish_metrics(send_to_telegram: true)` больше не требуется
   - Метрики сохраняются в файл `fastlane/metrics.json` для истории

---

## 📝 Примечания о режиме прогресс-сообщения

1. **Сохранение деталей:**
   - Детали каждого этапа автоматически сохраняются в файл `fastlane/.telegram_stages_details.json`
   - Детали накапливаются: после каждого этапа добавляются новые, старые остаются видимыми
   - В финальном сообщении видны детали всех завершенных этапов

2. **Передача между job'ами:**
   - Состояние прогресс-сообщения сохраняется в файлы для передачи между job'ами в GitLab CI
   - Необходимо настроить артефакты в `.gitlab-ci.yml` для передачи файлов между этапами

3. **Автоматическое обновление:**
   - Сообщение автоматически обновляется при переходе к следующему этапу
   - Прогресс-бар показывает выполненные, текущий и предстоящие этапы
   - Детали этапов дополняются по мере выполнения
